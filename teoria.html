wtf: html

Budowa znacznika HTML: <znacznik atrybut=”wartość”>treść</znacznik>

Semantyczny HTML – mający znaczenie – tagi HTML nadają znaczenie, dlatego nazywane są też znaczikami.

  • <!DOCTYPE html> Mówi o tym jakiego typu jest to dokument. W dawnych czasach definiowało się dokładny standard HTML. Obecnie nie należy wpisywać niczego więcej.
  • <html lang="en"> Należy tutaj umieścić kod języka, w którym jest strona. Przykładowo strona będzie czytana w tym języku przez asystenta głosowego dla osób niewidomych.
  • <main></main> Główna część dokumentu. Wewnątrz znacznika powinna zostać zawarta główna część dokumentu.
  • <section></section> Sekcja dokumentu. Część logicznie podzielonego dokumentu.
  • <article></article> Artykuł. Może istnieć samodzielnie i zawiera większą treść.
  • <header></header> Nagłówek. Opisuje nagłówek dokumentu / sekcji / artykułu.
  • <footer></footer> Stopka. Zawiera stopkę dokumentu / sekcji.
  • <p></p> Paragraf. Akapit.

Tekstowe nagłówki tytułowe. Im mniejszy numer ma znacznik nagłówka, tym większa jego ważność:

  • <h1></h1>
  • <h2></h2>
  • <h3></h3>
  • <h4></h4>
  • <h5></h5>
  • <h6></h6>

Dobrą praktyką jest manualne domykanie znaczników.
Każda sekcja powinna mieć tekstowy nagłówek tytułowy.
Doskonałym sposobem na sprawdzenie poprawności pliku .html jest strona: validator.w3.org

wtf: html ii

Pozostałe znaczniki:

  • <br> Nowa linia. Nie należy używać tego znacznika kilkukrotnie po sobie.
  • <b> Pogrubienia tekst (efekt wizualny).
  • <i> Nadaje kursywę tekstowi (efekt wizualny).
  • <strong> Podkreśla wysokią istotność tekstu.
  • <em> Podkreśla, że tekst może mieć inne znaczenie.
  • <a href="" target="">opis łącza</a> Linki / łącza wewnątrz dokumentu.
  • <nav> Nawigacja.
  • <img/> Obrazek o podanym adresie.

Niektóre atrybuty:

  • id Identyfikuje element. Przydaje się do nawigacji w obrębie dokumentu.
  • src Źródło (obrazka, pliku .js, etc.).
  • alt Tekst alternatywny. Informuje wyszukiwarkę o zawartości obrazka. Wykorzystywany również przez syntezatory mowy dla niewidomych.

Aby przejść na stronie do elementu o określonym 'id', należy atrybutowi 'href' znacznika 'a' nadać wartość: '#id_elementu_docelowego'.
W nazwach plików nie należy używać dużych liter, polskich znaków i spacji (w celu oddzielenia słów należy użyć myślnika).
Elementy powtarzalne na wielu podstronach, należy umieszczać poza znacznikiem 'main' (jedynie treść znajdująca się wewnątrz znacznika 'main' jest brana pod uwagę przez analizatory).
Atrybury pozwalają na identyfikowanie elementów (id), grupowanie elementów (class) lub nadanie im dodatkowych właściwości (href, src, alt). Lista globalnych atrybutów na MDN web docs.

opowieść o wężu, kebabie, wielbłądzie i Pascalu

Nazywanie rzeczy:

  • Nazwy zapisujemy bez polskich znaków.
  • Nie używamy spacji.
  • Używamy języka angielskiego.
  • Trzymamy się jednego schematu.

Sposoby zastąpienia spacji w nazewnictwie (dwie pierwsze wykorzystywane są najczęściej):

  • camelCase
  • kebab-case
  • snake_case
  • PascalCase

wtf: css ii

Specyficzność w CSS
Szeregując selektory pod względem ważności można oprzeć się o klasyfikację medalową, gdzie odpowiednio poszczególnymi kolorami medalów są następujące style:

  • !important (niezalecane)
  • style="" (niezalecane)
  • #name
  • .name
  • element

W przypadku równej ważności stylów, ważniejszy będzie ten, który jest niżej w kodzie, ponieważ kod jest interpretowany od góry do dołu.
Powinno się dodawać klasę do wszystkich elementów na stronie oraz unikać zagnieżdżeń.
Zasady nazewnictwa klas:

  • Używaj pojedynczych angielskich słów.
  • Nazywaj rzeczy zgodnie z ich funkcją na stronie.
  • Nazwy klas zaczynaj małą literą.
  • Nie zaczynaj od cyfry.
  • Skorzystaj z kebab case, kiedy musisz użyć większej ilości słów.
  • Staraj się, aby element miał jedną klasę.

jak działa Internet i przeglądarka

Zawartość URL (Uniform Resource Locator):

  • protokół
  • host (adres serwera)
  • port (ukryty)
  • root
  • ścieżka dostępu do zasobu
  • parametry (czasami ukryte)

kody odpowiedzi HTTP:

  • 1xx: należy poczekać na odpowiedź.
  • 2xx: OK.
  • 3xx: przekierowanie, należy szukać zasobu gdzie indziej.
  • 4xx: zasób nie został znaleziony.
  • 5xx: błąd serwera.

Protokół obsługujący strony internetowe to HTTP (coraz częściej HTTP/2)
Komunikacja między przeglądarką a serwerem odbywa się na zasadzie request - response.
Adres do zasobu bez '/' z przodu powoduje, że adres ten staje się adresem względnym i przeglądarka szuka pliku w aktualnym katalogu. Nazleży pamiętać o umieszczaniu '/' przed ścieżką do zasobu w przypadku gdy pracujemy z wykorzystaniem serwera.

wtf: javascrips

Co to jest ECMAScript 6+

  • ECMAScript jest to ustandaryzowany przez organizację ECMA skryptowy język programowania.
  • JavaScript implementuje rzeczy ze specyfikacji ECMAScript.
  • Rozwój ECMAScript mocno przyspieszył przez szerokie wykorzystanie JavaScriptu.
  • ES6 (i nowsze) wprowadzają wiele udogodnień dla developerów.
  • Nowe feature'y mogą nie być wspierane we wszystkich przeglądarkach, ale są na to sposoby.

Składowe JavaScript:

  • Wartości
  • Operacje porównania
  • Zmienne
  • Literały
  • Funkcje
  • Obiekty

Skrypty należy umieszczać na końcu znacznika 'body'.
Kod JS uruchamia się raz (po załadowaniu strony).
JavaScript to frontendowe programowanie.

jest git!

System kontroli wersji GIT dostępny jest na tej stronie.
Jeden projekt = jedno repozytorium!
Source Control / GitLens to narzędzia ułatwiające pracę z GITem w Visual Studio Code.
Po zainstalowaniu należy skonfigurować użytkownika wykonując polecenia:

  • git config --global user.name "Imię Nazwisko"
  • git config --global user.email adres_email

Dane użytkownika można sprawdzić z użyciem poleceń:

  • git config --get user.name
  • git config --get user.email

Polecenia GITa:

  • git init Inicjalizacja repozytorium (należy być w katalogu z projektem).
  • git status Sprawdzenie stan repozytorium (śledzone / nieśledzone pliki, ostatni commit, aktualną gałąź).
  • git add . Dodanie plików z aktualnego katalogu oraz podkatalogów jako śledzone.
  • git commit -m "komentarz" Zapis plików. Utworzenie commita ('wersji' projektu).
  • gitk Wyświetlenie historii commitów.

GIT podsumowanie:

  • GIT pozwala na sprawne zarządzanie zmianami w projekcie.
  • Działa lokalnie.
  • Do działania zdalnego potrzebuje serwera (np. GitHub).
  • Commity należy wykonywać po wykodaniu widocznej zmiany w projekcie (lepiej za często niż za rzadko).
  • Podstawowy flow: git add . / git commit -m "komentarz" / git push

życie w pudełku: display, wymiary i overflow

<div> Nie ma znaczenia semantycznego. Używamy go tylko w celu stylowania, gdy nie istnieje odpowiedni semantyczny znacznik.

Reguły / właściwości CSS:

  • display: block Powoduje, że elementy zajmują całą szerokość rodzica i ustawiają się jeden pod drugim.
  • display: inline Powoduje, że elementy zajmują na szerokość tyle miejsca ile potrzebują i ustawiają się obok siebie.
  • display: inline-block Powoduje, że elementy ustawiają się obok siebie, jednak otrzymują właściwości charakterystyczne dla elementów blokowych, np. 'width', 'height'.
  • min-width Ustala minimalną szerokość elementu.
  • min-height Ustala minimalną wysokość elementu.
  • max-width Ustala maksymalną szerokość elementu.
  • max-height Ustala maksymalną wysokość elementu.
  • overflow Steruje widocznością treści znajdującą się poza elementem.

życie w pudełku: jednostki

Jednostki służące do zapisu wymiarów w CSS:

  • 0 Nie wymaga dopisywania jednostki.
  • px Piksele.
  • % Procenty. Liczone są względem rodzica.
  • vw Viewport width. Szekokość względem szerokości okna.
  • vh Viewport height. Wysokość względem wysokości okna.

życie w pudełku: box model (padding / border / margin)

Box model - elementy:

  • padding Margines wewnętrzny.
  • border Obramowanie elementu oraz paddingu. Wchodzi w skład elementu.
  • margin Margines zewnętrzny. Informuje o odległościach, które muszą być zachowane od elementu. Działa jeynie dla wartości blokowych.

W przypadku powyższych własności (oraz innych kierunkowych) można nadać wartość jedynie dla jednej z krawędzi, np. 'margin-bottom'.
W celu nadania wartości dla wszystkich krawędzi, można wykorzystać zapis skrótowy (shorthand) i podać je odpowiednio dla krawędzi górnej, prawej, dolnej i lewej, np. 'padding: 20px 10px 0 10px;'.
Jeśli zostaną podane jedynie dwie wartości, pierwsza będzie się odnosić do krawędzi górnej i dolnej, a druga do lewej i prawej, np. 'margin: 10px 30px;'.
Jeśli zostanie podana jedynie jedna wartość, będzie się ona odnosić do wszystkich krawędzi, np.'padding: 5px;'.
Zapis skrótowy dla własności 'border' może mieć również postać, gdzie jako wartości podajemy kolejno: szerokość, styl i kolor, np. 'border: 2px solid #000;'.
Własność 'border-radius' umożliwia zaokrąglenie narożników poprzez podanie promienia zaokrąglenia (najczęściej w pikselach lub procentach).
Margines automatyczny zadziała tylko wtedy, gdy blokowy element będzie miał nadaną szerokość. W innym wypadku element zjmie całą szerokość rodzica i nie będzie już miejsca na umieszczenie marginesu po obu stronach elementu.

życie w pudełku: box-sizing

Właściwość 'box-sizing' decyduje o tym, co wchodzi w skład zdefiniowanych wymiarów elementu:

  • content-box Wartość domyślna. W skład zdefiniowanych wymiarów elemetnu wchodzi jedynie jego zawartość, a ewentualny padding i obramowanie są doliczane do tej wartości, powodując powiększenie elementu.
  • border-box W skład zdefiniowanych wymiarów elemetnu wchodzi zarówno zawartość, padding i obramowanie. Pozwala to na zdefiniowanie wymiarów elementu, gdzie w przypadku ewentualnego paddingu i obramowania zawartość zostanie odpowiednio pomniejszona, co nie spowoduje powiększenia elementu i będzie on miał nadal zdefiniowane wymiary.
