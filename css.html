wtf: css ii

Specyficzność w CSS
Szeregując selektory pod względem ważności można oprzeć się o klasyfikację medalową, gdzie odpowiednio poszczególnymi kolorami medalów są następujące style:

  • !important (niezalecane)
  • style="" (niezalecane)
  • #name
  • .name
  • element

W przypadku równej ważności stylów, ważniejszy będzie ten, który jest niżej w kodzie, ponieważ kod jest interpretowany od góry do dołu.
Powinno się dodawać klasę do wszystkich elementów na stronie oraz unikać zagnieżdżeń.
Zasady nazewnictwa klas:

  • Używaj pojedynczych angielskich słów.
  • Nazywaj rzeczy zgodnie z ich funkcją na stronie.
  • Nazwy klas zaczynaj małą literą.
  • Nie zaczynaj od cyfry.
  • Skorzystaj z kebab case, kiedy musisz użyć większej ilości słów.
  • Staraj się, aby element miał jedną klasę.

Strona dobierająca pasujące do siebie kolory.

życie w pudełku: display, wymiary i overflow

<div> Nie ma znaczenia semantycznego. Używamy go tylko w celu stylowania, gdy nie istnieje odpowiedni semantyczny znacznik.

Reguły/właściwości CSS:

  • display: block Powoduje, że elementy zajmują całą szerokość rodzica i ustawiają się jeden pod drugim.
  • display: inline Powoduje, że elementy zajmują na szerokość tyle miejsca ile potrzebują i ustawiają się obok siebie.
  • display: inline-block Powoduje, że elementy ustawiają się obok siebie, jednak otrzymują właściwości charakterystyczne dla elementów blokowych, np. 'width', 'height'.
  • min-width Ustala minimalną szerokość elementu.
  • min-height Ustala minimalną wysokość elementu.
  • max-width Ustala maksymalną szerokość elementu.
  • max-height Ustala maksymalną wysokość elementu.
  • overflow Steruje widocznością treści znajdującą się poza elementem.

życie w pudełku: jednostki

Jednostki służące do zapisu wymiarów w CSS:

  • 0 Nie wymaga dopisywania jednostki.
  • px Piksele.
  • % Procenty. Liczone są względem rodzica.
  • vw Viewport width. Szekokość względem szerokości okna.
  • vh Viewport height. Wysokość względem wysokości okna.

życie w pudełku: box model (padding/border/margin)

Box model - elementy:

  • padding Margines wewnętrzny.
  • border Obramowanie elementu oraz paddingu. Wchodzi w skład elementu.
  • margin Margines zewnętrzny. Informuje o odległościach, które muszą być zachowane od elementu. Działa jeynie dla wartości blokowych.

W przypadku powyższych własności (oraz innych kierunkowych) można nadać wartość jedynie dla jednej z krawędzi, np. 'margin-bottom'.
W celu nadania wartości dla wszystkich krawędzi, można wykorzystać zapis skrótowy (shorthand) i podać je odpowiednio dla krawędzi górnej, prawej, dolnej i lewej, np. 'padding: 20px 10px 0 10px;'.
Jeśli zostaną podane jedynie dwie wartości, pierwsza będzie się odnosić do krawędzi górnej i dolnej, a druga do lewej i prawej, np. 'margin: 10px 30px;'.
Jeśli zostanie podana jedynie jedna wartość, będzie się ona odnosić do wszystkich krawędzi, np.'padding: 5px;'.
Zapis skrótowy dla własności 'border' może mieć również postać, gdzie jako wartości podajemy kolejno: szerokość, styl i kolor, np. 'border: 2px solid #000;'.
Własność 'border-radius' umożliwia zaokrąglenie narożników poprzez podanie promienia zaokrąglenia (najczęściej w pikselach lub procentach).
Margines automatyczny zadziała tylko wtedy, gdy blokowy element będzie miał nadaną szerokość. W innym wypadku element zjmie całą szerokość rodzica i nie będzie już miejsca na umieszczenie marginesu po obu stronach elementu.

życie w pudełku: box-sizing

Właściwość 'box-sizing' decyduje o tym, co wchodzi w skład zdefiniowanych wymiarów elementu:

  • content-box Wartość domyślna. W skład zdefiniowanych wymiarów elemetnu wchodzi jedynie jego zawartość, a ewentualny padding i obramowanie są doliczane do tej wartości, powodując powiększenie elementu.
  • border-box W skład zdefiniowanych wymiarów elemetnu wchodzi zarówno zawartość, padding i obramowanie. Pozwala to na zdefiniowanie wymiarów elementu, gdzie w przypadku ewentualnego paddingu i obramowania zawartość zostanie odpowiednio pomniejszona, co nie spowoduje powiększenia elementu i będzie on miał nadal zdefiniowane wymiary.

życie w pudełku: position

Właściwość 'position' określa sposób pozycjonowania elementu na stronie:

  • static Wartość domyślna.
  • relative Tworzy z elementu układ odniesienia dla elementów posiadających wartość tej własności ustawioną na 'absolute'.
  • absolute Pozwala na nadawanie elementowi konkretnej pozycji wraz z wykorzystaniem własności: 'top', 'right', 'bottom', 'left'.
  • fixed Działa na podobnej zasadzie jak 'absolute', jednak element jest pozycjonowany względem ekranu a nie względem dokumentu (w czasie scrollowania nie zmienia swojej pozycji).
  • sticky Pozycja elementu zmienia się względem rodzica (który nie może posiadać właściwości 'overflow') wraz ze scrollowaniem, jednak nie wychodzi on poza obszar rodzica, lecz przykleja się do niego, kiedy rodzic przestaje być widoczny.
  • z-index Pozwala na sterowanie widocznością elementów nachodzących na siebie. Im większa wartość własności, tym jest on wyżej na osi Z.

życie w pudełku: float

Właściwość 'float' służy do opływania elementów:

  • left Element znajduje się z lewej strony, a treść opływa go z prawej.
  • right Element znajduje się z prawej strony, a treść opływa go z lewej.

Element z właściwością 'float' nie jest ograniczony przez rodzica. Aby nie wykraczał poza granice rodzica, można dodać rodzicowi na końcu pusty element posiadający właściwość 'clear', która dostosuje rozmiar rodzica do pływającego elementu.
Drugą metodą jest dodanie rodzicowi reguły 'overflow: hidden', jednak należy pamiętać, że określenie wysokości rodzica może spowodować przysłonięcie pewnej części pływającego elementu.
Właściwości 'float' nie należy stosować do budowania wyglądu strony.

css: flexbox

Właściwości CSS Flexbox nadawane kontenerowi:

  • display Służy do zmiany systemu rozmieszczania i pozycjonowania elementów.
  • flex-direction Odpowiada za zawijanie wiersza/kolumny.
  • flex-wrap Ustala główną oś CSS Flexbox.
  • flex-flow Skrót zawierający: 'flex-direction' i 'flex-wrap'.
  • justify-content Ustawia elementy wzdłuż osi głównej CSS Flexbox.
  • align-content Ustawia elementy wzdłuż osi prostopadłej do osi głównej CSS Flexbox.
  • align-items Ustawia elementy wzdłuż osi prostopadłej do osi głównej CSS Flexbox, ale w obrębie pojedynczego wiersza.

Właściwości CSS Flexbox nadawane elementom:

  • align-self Ustawia element wzdłuż osi prostopadłej do osi głównej CSS Flexbox.
  • order Ustala kolejość elementu.
  • flex-grow Odpowiada za wzrost elementu w celu wypełnienia pozostałego wolnego miejsca.
  • flex-shrink Odpowiada za kurczenie się elementu.
  • flex-basis Minimalny rozmiar bazowy elementu wewnątrz kontenera. Określa szerokość elementu - kiedy główną osią CSS Flexbox jest oś pozioma lub wysokość elementu - kiedy główną osią CSS Flexbox jest oś pionowa.
  • flex Skrót zawierający: 'flex-grow', 'flex-shrink' i 'flex-basis'.

normalizacja vs reset

Reset:

  • Przeglądarka posiada domyślne style 'user agent stylesheet'.
  • Reset wyłącza wszystkie domyślne style przeglądarki.
  • Selektorem uniwersalnym, wybierającym wszystkie elementy na stronie jest '*'. Selektor ten jest najczęściej wykorzystywany podczas resetu.
  • Nie należy wyłączać domyślnej właściwości 'outline', ponieważ ułatwia ona nawigację po stronie osobom niepełnosprawnym. Reset nie jest zatem dobrym rozwiązaniem i należy go unikać.

Normalizacja:

  • Normalizacja usprawnia wyświetlanie elementów pomiędzy różnymi przeglądarkami, ponieważ style domyślne nie są identyczne dla każdej z nich.
  • Powstał specjalny plik normalizujący, który można dodać do strony (oraz każdej z podstron). Nnajlepiej dodać go jako osobny plik 'normalize.css' przed plikiem z własnymi stylami.
  • Normalizacja to bardzo dobra alternatywa dla niezalecanego resetu.

Najlepszą praktyką jest miks: normalizacja oraz dodanie własnego resetu:

  • Dodaj plik 'normalize.css' do strony (oraz każdej z podstron).
  • Na górze pliku z własnymi stylami dodaj dla elementu html regułę box-sizing: border-box, a dla elementów *, *:before oraz *:after regułę box-sizing: inherit.

tła

Skrótowa właściwość CSS związana z tłem:

  • background Zawiera 8 poniższych właściwości szczegółowych. Nadpisuje wszystkie właściwości szegółowe zdefiniowane powyżej niej dla danego elementu. Zaleca się używanie właściwości szczegółowych zamiast właściwości skrótowej ze względu na większą czytelność kodu.

Szczegółowe właściwości CSS związane z tłem:

  • background-image Pozawala zaaplikować tło z poziomu stylów. Ścieżkę do obrazka należy podać względem pliku CSS.
  • background-position Ustawia tło względem elementu, który je dostaje.
  • background-repeat Pozwala na sterowanie powtarzaniem tła.
  • background-attachment Pozwala na sterowanie tłem podczas scrollowania strony.
  • background-origin Definiuje w którym miejscu tło powinno się zaczynać.
  • background-clip Definiuje w którym miejscu tło powinno być cięte.
  • background-color Definiuje kolor tła.
  • background-size Definiuje wielkość tła.

Przejścia między kolorami można utworzyć z wykorzystaniem gradientów. Na stronie cssgradient.io można skorzystać z gotowych gradientów jak również stworzyć własne.

css: grid

CSS Grid jest to oparty na siatce prostokątów sposób pozycjonowania elementów na stronie.
Pozycjonowanie tego typu działa jedynie na elementy znajdujące się bezpośrednio w kontenerze.
CSS Grid posiada następujące elementy:

  • Grid Line Linia gridowe. Linie są numerowane od wartości 1.
  • Grid Track (Column) Pojedyncza kolumna.
  • Grid Track (Row) Pojedynczy rząd.
  • Grid Cell Pojedyncza, prostokątna komórka.
  • Grid Area Prostokątny obszar składający się z wielu komórek.

Właściwości CSS Grid nadawane kontenerowi:

  • display Służy do zmiany systemu rozmieszczania i pozycjonowania elementów.
  • grid-template-columns Tworzy kolumny.
  • grid-template-rows Tworzy wiersze.
  • column-gap Tworzy odstępy między kolumnami.
  • row-gap Tworzy odstępy między wierszami.
  • grid-template-areas Określa nazwy obszarów siatki. Jako wartość należy podać nazwy obszarów (każdy wiersz zdefiniowany w osobnym cudzysłowie, zawierający tyle nazw obszarów, ile wynosi liczba kolumn). Pojedynczy obszar może przybierać jedynie kształt prostokąta (tyle samo kolumn w każdym wierszu dla konkretnej nazwy obszaru). Zamiast nazwy obszaru można podać '.', co stworzy nienazwany obszar, który może zostać wykorzystany np. do zdefiniowania odstępu między wierszami / kolumnami.

Właściwości CSS Grid nadawane elementom:

  • grid-area Przypisanie nazwy obszaru zdefiniowanej we właściwości 'grid-template-areas'. Nazwę należy podać bez cudzysłowu.
  • grid-row-start Rząd początkowy obszaru. Jako argument może przyjąć numer wiersza lub nazwę zdefiniowaną podczas nadawania wysokości wierszom (przed wartością w nawiasie kwadratowym).
  • grid-row-end Rząd końcowy obszaru. Jako argument może przyjąć numer wiersza, nazwę zdefiniowaną podczas nadawania wysokości wierszom (przed wartością w nawiasie kwadratowym) lub słowo 'span' oraz liczbę mówiącą ile kolejnych rzędów ma obejmować obszar.
  • grid-column-start Kolumna początkowa obszaru. Jako argument może przyjąć numer kolumny lub nazwę zdefiniowaną podczas nadawania szerokości kolumnom (przed wartością w nawiasie kwadratowym).
  • grid-column-end Kolumna końcowa obszaru. Jako argument może przyjąć numer kolumny, nazwę zdefiniowaną podczas nadawania szerokości kolumnom (przed wartością w nawiasie kwadratowym) lub słowo 'span' oraz liczbę mówiącą ile kolejnych rzędów ma obejmować obszar.
  • grid-column-end Kolumna końcowa obszaru. Jako argument może przyjąć numer kolumny, nazwę zdefiniowaną podczas nadawania szerokości kolumnom (przed wartością w nawiasie kwadratowym) lub słowo 'span' oraz liczbę mówiącą ile kolejnych rzędów ma obejmować obszar.
  • grid-area Skrót zawierający odpowiednio: 'grid-row-start', 'grid-column-start', 'grid-row-end', 'grid-column-end'. Każdą z wartości należy oddzielić znakiem '/'.
  • grid-row Skrót zawierający odpowiednio: 'grid-row-start', 'grid-row-end'. Każdą z wartości należy oddzielić znakiem '/'.
  • grid-column Skrót zawierający odpowiednio: 'grid-column-start', 'grid-column-end'. Każdą z wartości należy oddzielić znakiem '/'.

Jednostka 'fr' działa jak ułamek, gdzie licznikiem jest ilość 'fr' konkretnego elementu a mianownikiem suma 'fr' wszystkich elemetnów. Ułamek ten pokazuje jaką część wolnej przestrzeni zajmie dany element.
Funkcą 'repeat' pozwala na natychmiastowe stworzenie większej ilości kolumn/wierszy. Jako argumenty przyjmuje ilość wystąpień oraz wielkość danego elementu.
Funkcja 'minmax' pozwala na określenie minimalnego i maksymalnego rozmiaru kolumny/wiersza. Jako argumenty przyjmuje minimalny i maksymalny rozmiar obiektu.
CSS Grid nie zastępuje CSS Flexbox, lecz są dla siebie uzupełnieniem. CSS Grid powinien być wyorzystywany do budowania całych layoutów (kiedy na stronie występują wiersze, kolumny), natomiast CSS Flexbox do pozycjonowania elementów wewnątrz pojedynczego elementu.

css: media queries & mobile first

Responsiwe Web Design - projektowanie stron responsywnych (dostosowanych do ekranów o różnych wielkościach/formatach).
Media queries:

  • Słowo kluczowe @media pozwala określić w jakich przypadkach zastosowane zostaną konkretne style.
  • Media type wskazują ekran, urządzenie na którym zastosowane zostaną konkretne style, np. screen, print, all.
  • Media feature wskazują warunki dla których zastosowane zostaną konkretne style, np. min-width, max-width, min-height, max-height, color, ascpect-ratio, orientation.
  • Selektory zawarte wewnątrz 'media queries' nie posiadają zwiększonej specyficzności, dlatego należy dodawać je poniżej wcześniej zdefiniowanego elementu.
  • Wykorzystując 'media queries' należy nadpisywać jedynie własności, które ulegają zmianie (a nie wszystkie własności danego elementu).

Podejście 'mobile-first' polega na nadaniu stylów obowiązujących na całej stronie włącznie z wersją mobilną, a następnie dopasowanie elementów strony do większych ekranów poprzez dodanie 'breakpointów' oraz stylów wewnątrz nich. Breakpoint to szerokość ekranu gdzie następuje zmiana stylów.
